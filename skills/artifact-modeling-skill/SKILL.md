# Artifact Modeling

Start with a light graph that is useful for scope and verification.

- Record artifact type, URI, metadata, and stable edges.
- Prefer known relationships such as tests, documents, depends_on, and validates.
- Leave uncertain edges out or mark them in metadata for later inspection.

Read [references/artifact-graph.md](references/artifact-graph.md) for the graph shape.
